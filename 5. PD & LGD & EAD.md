# Credit Risk Modeling: PD, LGD, and EAD

---

## 1. Probability of Default (PD) Model

**Purpose:** Estimates the likelihood that a borrower will default within a given time frame (e.g., 12 months).

### Dependent Variable (Target)

* **Default Indicator (Binary: 0 or 1)**

  * **1 → Borrower defaulted**
  * **0 → Borrower did not default**

### Independent Variables (Predictors)

#### Borrower Characteristics

* Age
* Income
* Employment type
* Credit history

#### Loan Features

* Loan amount
* Interest rate
* Tenure

#### Behavioral Variables

* Missed payments
* Credit utilization
* Recent inquiries

#### Macroeconomic Factors

* GDP growth
* Inflation
* Unemployment rate

### Simple Example

If a bank studies past borrowers and sees that people with **low income and many missed payments** often default, the model learns this pattern.

Example prediction:

```
Borrower A
Income: Low
Missed payments: 3
Credit score: Poor

PD Model Output = 0.72

Meaning → 72% probability that the borrower may default within 12 months.
```

## 2. Loss Given Default (LGD) Model

**Purpose:** Estimates the percentage of the loan that will be lost if the borrower defaults.

### Dependent Variable (Target)

**LGD (%)**

```
LGD = Loss after Recovery / Exposure at Default
```

Values range from **0% to 100%**.

* **0% → Bank recovered everything**
* **100% → Bank recovered nothing**

### Independent Variables (Predictors)

#### Collateral Information

* Type of collateral
* Loan-to-value (LTV) ratio

#### Loan Characteristics

* Secured vs unsecured loan
* Loan amount
* Interest rate

#### Borrower Characteristics

* Credit score
* Financial health indicators

#### Macroeconomic Conditions

* Housing market trends
* Recovery rates

### Simple Example

A borrower takes a **₹10,00,000 home loan**.

After default, the bank sells the collateral house and recovers **₹7,00,000**.

```
Loss = 10,00,000 - 7,00,000 = 3,00,000

LGD = 3,00,000 / 10,00,000

LGD = 0.30 = 30%
```

Meaning → The bank lost **30% of the loan amount**.

## 3. Exposure at Default (EAD) Model

**Purpose:** Estimates the total exposure a bank will have at the time of default.

### Dependent Variable (Target)

**EAD (Monetary Value)**

```
EAD = Outstanding Balance + Potential Future Drawdowns
```

### Independent Variables (Predictors)

#### Loan and Credit Features

* Credit limit
* Utilization rate
* Loan type

#### Borrower Behavior

* Past usage patterns
* Repayment behavior

#### Macroeconomic Factors

* Interest rate trends
* Economic downturn risk

### Simple Example

A borrower has a **credit card with ₹5,00,000 limit**.

Current usage = **₹3,00,000**.

Before default, the borrower may use another **₹50,000**.

```
EAD = 3,00,000 + 50,000

EAD = 3,50,000
```

Meaning → Bank exposure at the time of default is **₹3,50,000**.

---

# How PD, LGD, and EAD Work Together

Banks combine these three models to estimate **Expected Credit Loss (ECL)**.

```
ECL = PD × LGD × EAD
```

### Example

```
PD = 0.10 (10% chance of default)
LGD = 0.40 (40% loss if default happens)
EAD = ₹5,00,000

ECL = 0.10 × 0.40 × 5,00,000

ECL = ₹20,000
```

Meaning → The bank expects an average loss of **₹20,000** from this borrower.

# Summary Table

| Model     | Dependent Variable   | Key Independent Variables                                       |
| --------- | -------------------- | --------------------------------------------------------------- |
| PD Model  | Default (0 or 1)     | Credit score, loan amount, missed payments, economic conditions |
| LGD Model | % Loss Given Default | Collateral type, LTV ratio, recovery rate, loan features        |
| EAD Model | Amount at Default    | Credit limit, utilization rate, repayment history               |

---

# Key Takeaway

* **PD → Probability that borrower will default**
* **LGD → How much money the bank will lose if default happens**
* **EAD → Total exposure at the time of default**

Together these models allow banks to **quantify credit risk and estimate expected losses**.
