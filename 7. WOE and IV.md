# Weight of Evidence (WoE) and Information Value (IV)

---

## Introduction

Weight of Evidence (WoE) and Information Value (IV) are important concepts used in **credit risk modeling**, especially in **logistic regression models** used by banks to predict whether a customer will default on a loan.

These techniques help transform variables and measure how strongly a variable can predict **good vs bad customers**.

They are widely used in:

* Credit risk modeling
* Scorecard development
* Feature selection
* Banking analytics

---

# 1. What is Weight of Evidence (WoE)?

**Weight of Evidence (WoE)** measures how strongly a variable separates **good customers** from **bad customers**.

It compares the proportion of **good borrowers** to **bad borrowers** in each category or bin of a variable.

### Simple Idea

WoE answers this question:

> "Does this group contain more good customers or more bad customers?"

If a group contains more **good customers**, WoE becomes **positive**.

If a group contains more **bad customers**, WoE becomes **negative**.

---

# 2. Recap: Role of WoE in Prediction

In credit risk modeling, the goal is to **predict whether a borrower will default or not**.

Models like **Logistic Regression** work better when variables have a **linear relationship with the log odds of default**.

WoE helps achieve this by:

* Converting categorical variables into numerical values
* Making relationships more linear
* Handling missing values
* Improving model stability

Because of these advantages, WoE is widely used in **credit scorecards**.

---

# 3. Dependent Variable: Default Flag

In credit risk models, the **target variable** is usually called the **Default Flag**.

| Default Flag | Meaning                    |
| ------------ | -------------------------- |
| 0            | Good customer (No default) |
| 1            | Bad customer (Defaulted)   |

The goal of the model is to **predict this flag**.

WoE calculations use this target variable to determine the proportion of **good vs bad customers**.

---

# 4. Binning Variables for WoE Analysis

Continuous variables cannot directly use WoE.

They must first be divided into **bins (ranges)**.

### Example: Salary Variable

| Salary Range | Customers        |
| ------------ | ---------------- |
| 0 – 30K      | Low salary       |
| 30K – 60K    | Medium salary    |
| 60K – 100K   | High salary      |
| 100K+        | Very high salary |

Each range becomes a **bin** where we compare good vs bad customers.

---

# 5. Comparing Good vs Bad Customers by Salary

Example dataset:

| Salary Range | Good Customers | Bad Customers |
| ------------ | -------------- | ------------- |
| 0 – 30K      | 40             | 60            |
| 30K – 60K    | 80             | 40            |
| 60K – 100K   | 120            | 30            |
| 100K+        | 160            | 20            |

Totals:

* Total Good = 400
* Total Bad = 150

Now we compute proportions.

| Salary Range | % Good           | % Bad           |
| ------------ | ---------------- | --------------- |
| 0 – 30K      | 40 / 400 = 0.10  | 60 / 150 = 0.40 |
| 30K – 60K    | 80 / 400 = 0.20  | 40 / 150 = 0.27 |
| 60K – 100K   | 120 / 400 = 0.30 | 30 / 150 = 0.20 |
| 100K+        | 160 / 400 = 0.40 | 20 / 150 = 0.13 |

---

# 6. WoE Formula

The formula for Weight of Evidence is:

```
WoE = ln( %Good / %Bad )
```

Where:

* ln = Natural logarithm
* %Good = proportion of good customers
* %Bad = proportion of bad customers

---

# 7. Example WoE Calculation

### Salary: 0 – 30K

```
%Good = 0.10
%Bad = 0.40

WoE = ln(0.10 / 0.40)
WoE = ln(0.25)
WoE = -1.386
```

### Salary: 100K+

```
%Good = 0.40
%Bad = 0.13

WoE = ln(0.40 / 0.13)
WoE = ln(3.08)
WoE = 1.12
```

---

# 8. Interpreting WoE Values

| WoE Value    | Interpretation      |
| ------------ | ------------------- |
| Positive WoE | More good customers |
| Negative WoE | More bad customers  |
| Zero WoE     | Equal good and bad  |

Example:

| Salary Range | WoE    |
| ------------ | ------ |
| 0 – 30K      | -1.386 |
| 30K – 60K    | -0.30  |
| 60K – 100K   | 0.41   |
| 100K+        | 1.12   |

Interpretation:

* Low salary → higher default risk
* High salary → lower default risk

---

# 9. What is Information Value (IV)?

**Information Value (IV)** measures the **predictive power of a variable**.

It tells us:

> "How useful is this variable in predicting default?"

IV is calculated using WoE values.

---

# 10. IV Formula

```
IV = Σ ( %Good - %Bad ) × WoE
```

Where the sum is taken across all bins.

---

# 11. Example IV Calculation

| Salary Range | %Good | %Bad | WoE    | (%Good-%Bad)×WoE |
| ------------ | ----- | ---- | ------ | ---------------- |
| 0 – 30K      | 0.10  | 0.40 | -1.386 | 0.416            |
| 30K – 60K    | 0.20  | 0.27 | -0.30  | 0.021            |
| 60K – 100K   | 0.30  | 0.20 | 0.41   | 0.041            |
| 100K+        | 0.40  | 0.13 | 1.12   | 0.302            |

Total IV:

```
IV = 0.416 + 0.021 + 0.041 + 0.302
IV = 0.78
```

---

# 12. IV Example with Education

Example bins:

| Education    | Good | Bad |
| ------------ | ---- | --- |
| High School  | 60   | 40  |
| Graduate     | 150  | 30  |
| Postgraduate | 190  | 10  |

After calculating WoE and IV, suppose:

```
IV (Education) = 0.25
```

This means **education is a strong predictor of default**.

---

# 13. Why WoE and IV Matter in Modeling

WoE and IV are important because they help:

### 1. Variable Transformation

Convert categorical variables into numeric format.

### 2. Feature Selection

Select variables that actually help predict default.

### 3. Scorecard Modeling

Used heavily in **credit scorecards used by banks**.

### 4. Model Stability

WoE makes relationships smoother and easier for logistic regression.

---

# 14. Variable Reduction using IV

When building a model, we often start with **many variables**.

IV helps remove weak predictors.

Example:

| Variable       | IV   |
| -------------- | ---- |
| Salary         | 0.78 |
| Education      | 0.25 |
| Age            | 0.03 |
| Marital Status | 0.01 |

Variables like **Age and Marital Status** may be removed because they provide little predictive power.

---

# 15. IV Interpretation Scale

IV values are interpreted using standard thresholds.

| IV Value    | Predictive Power                  |
| ----------- | --------------------------------- |
| < 0.02      | Not useful                        |
| 0.02 – 0.10 | Weak predictor                    |
| 0.10 – 0.30 | Medium predictor                  |
| 0.30 – 0.50 | Strong predictor                  |
| > 0.50      | Suspicious / Possibly overfitting |

Example:

| Variable  | IV   | Meaning                             |
| --------- | ---- | ----------------------------------- |
| Salary    | 0.78 | Very strong (check for overfitting) |
| Education | 0.25 | Good predictor                      |
| Age       | 0.03 | Weak predictor                      |

---

# 16. Step-by-Step Process to Calculate WoE and IV

### Step 1

Choose a variable (example: Salary).

### Step 2

Create bins.

Example:

```
0–30K
30–60K
60–100K
100K+
```

### Step 3

Count good and bad customers.

### Step 4

Calculate:

```
%Good
%Bad
```

### Step 5

Compute WoE:

```
WoE = ln(%Good / %Bad)
```

### Step 6

Compute IV contribution:

```
(%Good - %Bad) × WoE
```

### Step 7

Add all IV contributions.

```
IV = Sum of all bins
```

WoE and IV are fundamental techniques used in **credit risk modeling, scorecard development, and banking analytics**.

They help transform data, evaluate variables, and build more interpretable predictive models.
