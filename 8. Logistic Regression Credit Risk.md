# Logistic Regression in Credit Risk Modeling

---

## 1. Introduction to Logistic Regression

Logistic Regression is one of the most important statistical models used in **banking, credit risk modeling, and machine learning**.

It is mainly used for **binary classification problems**, where the outcome has only **two possible values**.

Examples:

| Situation         | Possible Outcomes    |
| ----------------- | -------------------- |
| Loan repayment    | Default / No Default |
| Email filtering   | Spam / Not Spam      |
| Disease detection | Sick / Healthy       |
| Credit approval   | Approved / Rejected  |

In banking, Logistic Regression is commonly used to **predict whether a borrower will default on a loan or not**.

So instead of predicting a number, the model predicts a **probability**.

Example:

```
Probability of Default = 0.82
```

This means there is an **82% chance that the borrower will default**.

---

## 2. What is Default in Banking?

In banking, **Default** means that a borrower has **failed to repay the loan according to the agreed terms**.

Most banks follow the rule that:

**A loan is considered default if payment is overdue for more than 90 days.**

Example:

| Customer   | Loan EMI Due | Days Past Due | Default Status |
| ---------- | ------------ | ------------- | -------------- |
| Customer A | 1 Jan        | 20 days       | Not Default    |
| Customer B | 1 Jan        | 95 days       | Default        |
| Customer C | 1 Jan        | 120 days      | Default        |

So in modeling we create a **Default Flag variable**.

| Default Flag | Meaning                  |
| ------------ | ------------------------ |
| 1            | Customer Defaulted       |
| 0            | Customer Did Not Default |

This becomes the **Target Variable (Dependent Variable)** in credit risk models.

## 3. Historical Window Explained

Banks do not build models using random data. They use **historical loan data**.

The **Historical Window** refers to the **past time period used to train the model**.

Example:

A bank may collect **5 years of loan data**.

```
Historical Window = 2018 – 2023
```

During this period the bank records:

| Customer | Income | Loan Amount | Credit History | Default |
| -------- | ------ | ----------- | -------------- | ------- |
| C1       | 50,000 | 3,00,000    | Good           | 0       |
| C2       | 20,000 | 5,00,000    | Poor           | 1       |
| C3       | 60,000 | 2,00,000    | Good           | 0       |

Using this past data, the bank trains the model to **predict future defaults**.

---

## 4. Understanding Probability of Default (PD)

**Probability of Default (PD)** is the probability that a borrower will **fail to repay the loan within a given time period**.

Usually the time period is:

```
12 months
```

Example:

| Customer | PD Value | Interpretation        |
| -------- | -------- | --------------------- |
| A        | 0.02     | 2% chance of default  |
| B        | 0.25     | 25% chance of default |
| C        | 0.80     | 80% chance of default |

Banks use PD to:

* Measure credit risk
* Decide loan approvals
* Set interest rates
* Calculate Expected Credit Loss (ECL)

Example:

If

```
PD = 0.20
```

Then the borrower has **20% probability of defaulting** within the next year.

## 5. Regulatory Guidelines: Basel & IFRS9

Credit risk models used by banks must follow **international regulatory standards**.

Two important frameworks are:

| Framework     | Purpose                                                    |
| ------------- | ---------------------------------------------------------- |
| Basel Accords | Ensure banks maintain enough capital to absorb losses      |
| IFRS 9        | Accounting standard for calculating expected credit losses |

### Basel Framework

The Basel framework is created by the **Basel Committee on Banking Supervision (BCBS)**.

Its purpose is to ensure that banks:

* Maintain adequate capital
* Manage risk properly
* Avoid financial crises

Basel requires banks to estimate:

* Probability of Default (PD)
* Loss Given Default (LGD)
* Exposure at Default (EAD)

### IFRS 9

IFRS 9 requires banks to estimate **Expected Credit Loss (ECL)**.

Formula:

```
ECL = PD × LGD × EAD
```

Example:

| Component | Value    |
| --------- | -------- |
| PD        | 0.10     |
| LGD       | 0.50     |
| EAD       | 1,00,000 |

Calculation:

```
ECL = 0.10 × 0.50 × 100000
ECL = 5000
```

The bank expects an average loss of **₹5000**.

## 6. Logistic Regression & Sigmoid Function

Logistic Regression converts a linear equation into a **probability between 0 and 1**.

This is done using the **Sigmoid Function**.

Sigmoid formula:

```
P = 1 / (1 + e^-z)
```

Where:

```
z = β0 + β1X1 + β2X2 + ... + βnXn
```

The sigmoid function produces an **S‑shaped curve**.

Output range:

```
0 to 1
```

Example:

| Linear Score (z) | Probability |
| ---------------- | ----------- |
| -3               | 0.047       |
| 0                | 0.50        |
| 2                | 0.88        |
| 4                | 0.98        |

Interpretation:

* Higher score → Higher probability of default

---

## 7. Logistic Regression Equation Breakdown

The logistic regression model looks like this:

```
log(P / (1 - P)) = β0 + β1X1 + β2X2 + β3X3
```

This is called the **Logit Function**.

Where:

| Symbol | Meaning                |
| ------ | ---------------------- |
| P      | Probability of default |
| β0     | Intercept              |
| β1, β2 | Model coefficients     |
| X1, X2 | Input variables        |

Example:

```
log(P/(1-P)) = -2 + 0.03 × Age + 0.00002 × Income
```

If:

```
Age = 40
Income = 50000
```

Then:

```
z = -2 + (0.03×40) + (0.00002×50000)
```

```
z = -2 + 1.2 + 1
z = 0.2
```

Now apply sigmoid:

```
P = 1/(1+e^-0.2)
```

```
P ≈ 0.55
```

So the **Probability of Default = 55%**.

## 8. What are Dummy Variables?

Machine learning models work with **numbers**, not text categories.

So categorical variables must be converted into **Dummy Variables**.

Dummy variables are **binary variables (0 or 1)**.

Example:

Original variable:

| Employment Type |
| --------------- |
| Salaried        |
| Self Employed   |
| Business        |

Converted to dummy variables:

| Salaried | Self_Employed | Business |
| -------- | ------------- | -------- |
| 1        | 0             | 0        |
| 0        | 1             | 0        |
| 0        | 0             | 1        |

Another example:

| Gender |
| ------ |
| Male   |
| Female |

Converted:

| Male |
| ---- |
| 1    |
| 0    |

Dummy variables allow models to **use categorical information in numeric form**.

---

## 9. Why Numeric Variables Matter in Banking

Most machine learning models, including logistic regression, require **numeric inputs**.

Examples of numeric banking variables:

| Variable     | Example   |
| ------------ | --------- |
| Income       | 50000     |
| Loan Amount  | 300000    |
| Age          | 35        |
| Credit Score | 720       |
| Loan Tenure  | 24 months |

These numbers help the model **identify risk patterns**.

Example pattern:

| Income | Loan Amount | Default |
| ------ | ----------- | ------- |
| 20000  | 500000      | 1       |
| 80000  | 200000      | 0       |

The model may learn:

```
Low income + high loan amount → higher default risk
```

Therefore numeric variables are **critical for risk prediction**.
