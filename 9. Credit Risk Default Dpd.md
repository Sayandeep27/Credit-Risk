# Credit Risk Concepts: Default, DPD, Performance File, Default Events, Cure Period, Charge-off & Bankruptcy

---

## 1. Introduction

In **credit risk modeling**, banks and financial institutions need to evaluate whether a borrower will repay a loan or fail to repay it. The possibility that a borrower will not repay the loan is called **credit risk**.

To manage this risk, banks analyze borrower behavior using historical data. Some of the most important concepts used in credit risk analysis include:

* Default
* Days Past Due (DPD)
* Performance File
* Default Events
* Cure Period
* Episode Definition
* Charge-off
* Bankruptcy

Understanding these concepts is essential when building models such as:

* Probability of Default (PD)
* Credit Scorecards
* Expected Credit Loss (ECL)

This document explains these concepts in **simple language with practical examples**.

---

# 2. What is Default?

## Definition

In credit risk, **default** occurs when a borrower fails to meet the repayment obligation of a loan according to the agreed terms.

In simple words:

> Default means the borrower **does not repay the loan for a long enough period that the bank considers the loan risky or uncollectable**.

---

## Regulatory Definition

Under most regulatory frameworks (Basel guidelines), a borrower is considered **in default when:**

* The borrower is **90 days or more past due** on a payment, OR
* The bank believes the borrower is **unlikely to pay the loan fully**.

---

## Simple Example

| Customer | EMI Due Date | EMI Paid? | Days Past Due | Status     |
| -------- | ------------ | --------- | ------------- | ---------- |
| A        | 1 Jan        | Yes       | 0             | Good       |
| B        | 1 Jan        | Late      | 30            | Delinquent |
| C        | 1 Jan        | Not Paid  | 95            | Default    |

Explanation:

* Customer A paid on time → No issue
* Customer B paid late → Delinquency
* Customer C did not pay for 95 days → **Default**

---

## Why Default is Important

Banks track default because it helps them:

* Measure **credit risk**
* Calculate **Probability of Default (PD)**
* Estimate **Expected Credit Loss (ECL)**
* Decide **loan approval policies**

---

# 3. Understanding DPD (Days Past Due)

## Definition

**DPD (Days Past Due)** measures how many days a borrower is late in making a payment.

Formula:

```
DPD = Payment Date – Due Date
```

If the payment has not been made, DPD continues increasing daily.

---

## Example

| Due Date | Payment Date | DPD |
| -------- | ------------ | --- |
| 1 Jan    | 1 Jan        | 0   |
| 1 Jan    | 10 Jan       | 9   |
| 1 Jan    | 15 Feb       | 45  |

Explanation:

* If payment is made on due date → DPD = 0
* If payment is made late → DPD becomes positive

---

## Common DPD Buckets

Banks categorize delinquency using DPD ranges.

| DPD Range | Category          | Meaning                       |
| --------- | ----------------- | ----------------------------- |
| 0         | Current           | Payment on time               |
| 1–30      | Early delinquency | Slight delay                  |
| 31–60     | Moderate risk     | Payment delay increasing      |
| 61–90     | High risk         | Serious delay                 |
| 90+       | Default           | Borrower considered defaulted |

---

## Real Example

Suppose:

* EMI due = 1st January
* Customer pays on = 20th February

```
DPD = 50 days
```

This falls in the **31–60 DPD bucket**, meaning the borrower is showing **significant delinquency**.

# 4. Performance File Explained

## Definition

A **performance file** is a dataset that tracks how borrowers repay their loans over time.

It records the **monthly behavior of the loan after it is issued**.

---

## Why Performance File is Important

Banks use performance files to:

* Track borrower repayment behavior
* Identify delinquency trends
* Detect default events
* Train **credit risk models**

---

## Example Structure of Performance File

| Customer ID | Month | Loan Balance | DPD | Payment Made |
| ----------- | ----- | ------------ | --- | ------------ |
| 1001        | Jan   | 50,000       | 0   | Yes          |
| 1001        | Feb   | 48,000       | 5   | Yes          |
| 1001        | Mar   | 46,000       | 35  | Late         |
| 1001        | Apr   | 46,000       | 95  | No           |

Explanation:

* January → Payment on time
* February → Slight delay
* March → Serious delay
* April → Default (DPD > 90)

This **time-series data** is essential for building **PD models**.

# 5. Types of Default Events

A **default event** is any situation where the borrower is considered to have defaulted.

Common default triggers include:

---

## 1. 90+ DPD Default

If a borrower misses payments for **90 days or more**, the account is classified as default.

Example:

| Customer | DPD |
| -------- | --- |
| A        | 25  |
| B        | 65  |
| C        | 120 |

Customer C is considered **defaulted**.

---

## 2. Bankruptcy

If the borrower legally declares bankruptcy, the loan is considered defaulted.

Reason:

The borrower legally states they cannot repay debts.

---

## 3. Charge-off

When the bank decides the loan is **unlikely to be recovered**, it writes it off as a loss.

This is also considered a **default event**.

---

# 6. Cure Period

## Definition

A **cure period** is the time during which a borrower recovers from delinquency by making payments and bringing the account back to normal status.

---

## Example

| Month | DPD |
| ----- | --- |
| Jan   | 0   |
| Feb   | 35  |
| Mar   | 60  |
| Apr   | 0   |

Explanation:

* Customer became delinquent in Feb and Mar
* In April they paid all dues

This means the borrower **cured the delinquency**.

---

## Why Cure Period Matters

Banks use cure periods to determine:

* Whether the borrower has recovered
* Whether the default episode has ended

---

# 7. Episode Definition

## Definition

An **episode** is a continuous period during which a borrower remains delinquent.

It starts when delinquency begins and ends when the borrower cures the delinquency.

---

## Example

| Month | DPD |
| ----- | --- |
| Jan   | 0   |
| Feb   | 35  |
| Mar   | 65  |
| Apr   | 90  |
| May   | 0   |

Explanation:

* February → delinquency starts
* March → continues
* April → reaches default
* May → borrower pays dues

This entire period **Feb → Apr** is called **one delinquency episode**.

---

# 8. Charge-off

## Definition

A **charge-off** occurs when the bank officially writes off a loan as a loss because it believes the debt will not be collected.

However, the borrower **still legally owes the money**.

---

## Example

| Loan Amount | Outstanding | Bank Action |
| ----------- | ----------- | ----------- |
| $10,000     | $8,500      | Charged-off |

The bank removes the loan from its active assets but may still attempt recovery.

---

## When Charge-offs Happen

Usually after:

* Long delinquency (often **180 days or more**)
* Repeated collection failures

---

# 9. Bankruptcy

## Definition

**Bankruptcy** is a legal process where a borrower declares inability to repay debts.

A court supervises the resolution of debts.

---

## What Happens During Bankruptcy

Possible outcomes include:

* Debt restructuring
* Partial repayment
* Full discharge of debt

---

## Example

A borrower has:

* Credit card debt
* Personal loan
* Mortgage

If they file bankruptcy, the court may:

* Cancel some debts
* Allow repayment through a structured plan

For banks, bankruptcy is treated as a **default event**.
