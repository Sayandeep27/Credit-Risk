# Credit Risk Modeling Concepts

## Dependent Variable (Default Flag)

The dependent variable is the **target variable** used in a credit risk model. It indicates whether a borrower is a **good customer** or a **bad customer**.

| Value | Meaning                    |
| ----- | -------------------------- |
| 0     | Good customer (No default) |
| 1     | Bad customer (Default)     |

Example:

If a customer repays all loan EMIs on time → **Default Flag = 0**

If a customer stops paying EMIs → **Default Flag = 1**

---

## Estimated Credit Loss (ECL)

Estimated Credit Loss represents the **expected financial loss a bank may face if a borrower defaults**.

### Formula

```
ECL = PD × LGD × EAD
```

| Component | Meaning                |
| --------- | ---------------------- |
| PD        | Probability of Default |
| LGD       | Loss Given Default     |
| EAD       | Exposure at Default    |

### Example

Loan Amount = $10000

PD = 10% = 0.10

LGD = 40% = 0.40

EAD = $10000

```
ECL = 0.10 × 0.40 × 10000
ECL = 400
```

Expected loss = **$400**

---

## Bad Definition

Bad Definition determines **when a customer is classified as a defaulter**.

### Industry Standard

Most banks use **90+ Days Past Due (DPD)**.

| DPD   | Meaning                |
| ----- | ---------------------- |
| 0‑30  | Normal delay           |
| 30‑60 | Risk increasing        |
| 60‑90 | High risk              |
| 90+   | Default / Bad customer |

Example:

If EMI is unpaid for **95 days**, the customer becomes **Bad**.

## Python Implementation Example

Below is a simple example of preparing loan data using Python.

```python
import pandas as pd

try:
    df = pd.read_csv("loan_data.csv")
except FileNotFoundError:
    print("File not found")

# create default flag

```

Explanation:

* Read loan dataset
* Create **binary target variable**
* 1 represents default
* 0 represents non‑default

## Loan Status Categories

Loan datasets often contain multiple loan status values.

### Charge‑Off

A **charge‑off** occurs when the lender believes the debt will not be recovered.

Example:

Customer stops paying for long time → bank marks loan as **Charged Off**.

### Write‑Off

A **write‑off** is when the bank records the loan as an **operational loss in accounting**.

Difference:

| Term       | Meaning                  |
| ---------- | ------------------------ |
| Charge‑off | Recovery abandoned       |
| Write‑off  | Accounting loss recorded |

---

## Fine Classing

Fine classing is the process of **converting continuous variables into small buckets**.

Example:

Income Variable

| Income Range | Category |
| ------------ | -------- |
| 0‑20000      | Low      |
| 20000‑50000  | Medium   |
| 50000‑100000 | High     |

Purpose:

* Identify patterns
* Improve model interpretability

---

## Weight of Evidence (WoE)

Weight of Evidence measures **how well an independent variable predicts default**.

### Formula

```
WoE = ln(% of Good / % of Bad)
```

Example:

| Income Bucket | Good | Bad |
| ------------- | ---- | --- |
| Low           | 20   | 80  |
| Medium        | 50   | 50  |
| High          | 90   | 10  |

Interpretation:

* Positive WoE → lower risk
* Negative WoE → higher risk

---

## Coarse Classing

Coarse classing means **combining multiple fine classes into larger groups**.

Example:

Fine Classes:

| Income       | Group  |
| ------------ | ------ |
| 0‑10000      | Low    |
| 10000‑20000  | Low    |
| 20000‑40000  | Medium |
| 40000‑100000 | High   |

Coarse Classes:

| Income      | Group  |
| ----------- | ------ |
| 0‑20000     | Low    |
| 20000‑40000 | Medium |
| 40000+      | High   |

Purpose:

* Reduce complexity
* Improve stability of the model

# Summary

Credit risk modeling involves several key steps:

1. Define the **dependent variable (default flag)**
2. Calculate **Expected Credit Loss (ECL)** using PD, LGD, and EAD
3. Define **bad customers using 90+ DPD**
4. Use **multiple independent variables**
5. Prepare data using **Python and Pandas**
6. Understand **loan status categories**
7. Perform **fine classing**
8. Calculate **Weight of Evidence**
9. Perform **coarse classing**

These steps are fundamental in building **credit risk models used by banks and financial institutions**.
